## 三、最關鍵的技術重點（務必照做，避免踩坑）

**回測需要「單檔股票的歷史序列」，不是全市場當日資料。**

- ❌ 錯誤端點：`STOCK_DAY_ALL` → 只給全市場「今天」一筆，無法回測。
- ✅ 正確端點：`https://www.twse.com.tw/exchangeReport/STOCK_DAY?response=json&date=YYYYMMDD&stockNo=股票代號`
  - 回傳某檔股票「一整個月」的每日資料。
  - 回傳 JSON 結構：`{ "stat":"OK", "fields":[...], "data":[[...],[...]] }`
  - `fields` 順序為：日期、成交股數、成交金額、開盤價、最高價、最低價、**收盤價（第 7 欄，index 6）**、漲跌價差、成交筆數。
  - 收盤價字串含逗號（如 "572.0"、"1,005.0"），要去逗號轉 float。
  - 日期是民國格式（如 "113/09/02"）。
- 要回測一年，就跑 12 次（每月一次），把 `date` 換成各月份，再把每月的 data 串起來。
- **限流**：證交所每 5 秒最多 3 個 request，所以每次抓之間 `sleep(2)` 秒，否則會被擋。

我有一個股票機器人平台的第一期 MVP 要完成。這個資料夾裡已經有可用的回測引擎

- 提供一個 API：GET /api/prices?stock=2330&months=12
- 它去證交所抓該股票最近 N 個月的每日收盤價，回傳一個收盤價的陣列（由舊到新）。
- 證交所端點：https://www.twse.com.tw/exchangeReport/STOCK_DAY?response=json&date=YYYYMMDD&stockNo=股票代號
- 收盤價在每筆 data 的 index 6，字串要去逗號轉 float。
- 每次 request 之間 sleep 2 秒（證交所限流：5 秒最多 3 次）。
- 先用 stock=2330 實際跑一次，印出抓到幾天資料、最後一筆收盤價，確認真的抓到了。

【第二步：把後端接上回測引擎】
- 提供另一個 API：POST /api/backtest
  body 收 { "stock":"2330", "months":12, "bot": {買賣規則 JSON} }
- 流程：先用第一步抓股價 → 丟給 bot_engine.run_bot() → 回傳回測結果（交易次數、勝率、報酬、目前訊號、log）。
- 用 sample_bots.py 裡的「KD 抄底機器人」對 2330 實際跑一次，確認有回測結果。

【第三步：改前端原型接真實資料】

- 把後端回傳的結果顯示在原本的結果區。
- 保留原本的積木式組裝介面，不要動它的操作方式。

【第四步：加「儲存我的機器人」】

- 加一個「我的機器人」清單，能載入之前存的機器人回工廠。

【部署】
- 前端（index.html + engine.js）部署到 GitHub Pages。
- 後端部署到 Render（免費方案即可）。
- 注意前端要呼叫後端，記得處理 CORS（後端加 flask-cors 或對應設定）。
- 給我最後的網址，讓我能用手機打開測試。

重要原則：
- 平台只產生「參考訊號」，絕對不要做任何自動下單、不要接券商、不要碰金流。
- 每一步驟做完都先實際跑給我看結果，確認對了再繼續。
